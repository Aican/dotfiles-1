              <div class="highlight"><pre><div class="line" id="LC1">*pastie.txt*  Interface for pastie.org</div><div class="line" id="LC2"><br/></div><div class="line" id="LC3">Author:  Tim Pope &lt;vimNOSPAM@tpope.org&gt;         *pastie-author*</div><div class="line" id="LC4">License: Same terms as Vim itself (see |license|)</div><div class="line" id="LC5"><br/></div><div class="line" id="LC6">This plugin is only available if &#39;compatible&#39; is not set.</div><div class="line" id="LC7"><br/></div><div class="line" id="LC8">A working Ruby install is required (a Vim compiled with |if_ruby| support is</div><div class="line" id="LC9">not necessary).</div><div class="line" id="LC10"><br/></div><div class="line" id="LC11">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*pastie-:Pastie*</div><div class="line" id="LC12">:Pastie creates a new paste (example arguments shown below).  Use :w to save</div><div class="line" id="LC13">it by posting it to the server (the parser used is derived from the Vim</div><div class="line" id="LC14">filetype).  This updates the filename and stores the new url in the primary</div><div class="line" id="LC15">selection/clipboard when successful.  :Pastie!  creates a paste, saves, and</div><div class="line" id="LC16">closes the buffer, except when loading an existing paste.</div><div class="line" id="LC17"><br/></div><div class="line" id="LC18">:Pastie                   Create a paste from all open windows</div><div class="line" id="LC19">:Pastie!                  Create a paste from all open windows and paste it</div><div class="line" id="LC20">:1,10Pastie               Create a paste from the specified range</div><div class="line" id="LC21">:%Pastie                  Use the entire current file to create a new paste</div><div class="line" id="LC22">:Pastie foo.txt bar.txt   Create a paste from foo.txt and bar.txt</div><div class="line" id="LC23">:Pastie! foo.txt          Paste directly from foo.txt</div><div class="line" id="LC24">:Pastie a                 Create a paste from the &quot;a register</div><div class="line" id="LC25">:Pastie @                 Create a paste from the default (unnamed) register</div><div class="line" id="LC26">:Pastie *                 Create a paste from the primary selection/clipboard</div><div class="line" id="LC27">:Pastie _                 Create a new, blank paste</div><div class="line" id="LC28">:768Pastie                Load existing paste 768</div><div class="line" id="LC29">:0Pastie                  Load the newest paste</div><div class="line" id="LC30">:Pastie http://pastie.org/768            Load existing paste 768</div><div class="line" id="LC31">:Pastie http://pastie.org/123456?key=... Use login from pastie bot</div><div class="line" id="LC32"><br/></div><div class="line" id="LC33">Regardless of the command used, on the first write, this script will create a</div><div class="line" id="LC34">new paste, and on subsequent writes, it will update the existing paste.  If a</div><div class="line" id="LC35">bang is passed to a command that load an existing paste (:768), the first</div><div class="line" id="LC36">write will update as well.  If the loaded paste was not created in the same</div><div class="line" id="LC37">vim session, or with an account extracted from your Firefox cookies, updates</div><div class="line" id="LC38">will almost certainly silently fail.  (Advanced users can muck around with</div><div class="line" id="LC39">g:pastie_session_id if desired).</div><div class="line" id="LC40"><br/></div><div class="line" id="LC41">As hinted at earlier, pastie.vim will snoop around in your Firefox cookies,</div><div class="line" id="LC42">and use an account cookie if one is found.
